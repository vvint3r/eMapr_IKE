## KEY VISUAL MODELS

### 1. COMPETENCY HIERARCHY
```
┌─────────────────────────────────────────┐
│   GRADUATION COMPETENCIES (3-6)        │  ← Broad, cross-curricular
│   21st Century Skills                   │
└────────────────┬────────────────────────┘
                 │
┌────────────────┴────────────────────────┐
│   CONTENT COMPETENCIES (3-6 per area)   │  ← Discipline-specific
│   Subject-area expertise                │
└────────────────┬────────────────────────┘
                 │
┌────────────────┴────────────────────────┐
│   PERFORMANCE INDICATORS (5-10 each)    │  ← Measurable evidence

└────────────────┬────────────────────────┘
                 │

│   LEARNING OBJECTIVES (Multiple)        │  ← Lesson-level targets

└─────────────────────────────────────────┘
```

### 2. BACKWARD DESIGN PROCESS (Understanding by Design)
```
┌──────────────────────────────────────────────────────┐
│  STAGE 1: IDENTIFY DESIRED RESULTS                   │
│  • What should students know, understand, do?        │
│  • Define learning outcomes & competencies           │
└───────────────────┬──────────────────────────────────┘
                    ↓
┌──────────────────────────────────────────────────────┐
│  STAGE 2: DETERMINE ACCEPTABLE EVIDENCE              │
│  • How will students demonstrate mastery?            │
│  • Design assessments & rubrics                      │
└───────────────────┬──────────────────────────────────┘
                    ↓
┌──────────────────────────────────────────────────────┐
│  STAGE 3: PLAN LEARNING EXPERIENCES                  │
│  • What instruction will prepare students?           │
│  • Design activities, sequence, resources            │
└──────────────────────────────────────────────────────┘
```

### 3. BLOOM'S TAXONOMY (Revised 2001)
```
        ┌─────────────┐
        │   CREATE    │  ← Highest Order (Design, Construct)
        ├─────────────┤
        │  EVALUATE   │  ← (Critique, Judge)
        ├─────────────┤
        │   ANALYZE   │  ← (Compare, Organize)
        ├─────────────┤
        │    APPLY    │  ← (Implement, Execute)
        ├─────────────┤
        │ UNDERSTAND  │  ← (Explain, Summarize)
        ├─────────────┤
        │  REMEMBER   │  ← Lowest Order (Recall, Recognize)
        └─────────────┘
```

### 4. SOLO TAXONOMY (Depth of Understanding)
```
Level 1: PRESTRUCTURAL
  → Lacks understanding, misses the point

Level 2: UNISTRUCTURAL
  → Identifies one relevant aspect

Level 3: MULTISTRUCTURAL
  → Identifies multiple relevant aspects (disconnected)

Level 4: RELATIONAL
  → Integrates parts into coherent whole

Level 5: EXTENDED ABSTRACT
  → Generalizes to new domains, creates new ideas
```

```
┌──────────────────────────────────────────────────┐
│          LEARNING TASKS (Whole Tasks)            │
│    Authentic, meaningful, progressive complexity │
│                                                  │

│  │   SUPPORTIVE     │    │   PROCEDURAL     │  │
│  │   INFORMATION    │    │   INFORMATION    │  │
│  │                  │    │                  │  │
│  │ • Concepts       │    │ • Steps          │  │
│  │ • Mental models  │    │ • Rules          │  │
│  │ • Strategies     │    │ • Just-in-time   │  │
│  │ • Case studies   │    │ • Job aids       │  │
│  └──────────────────┘    └──────────────────┘  │
│                                                  │
│  ┌──────────────────────────────────────────┐  │
│  │     PART-TASK PRACTICE                   │  │
│  │  Drill routine aspects to automaticity   │  │
│  └──────────────────────────────────────────┘  │
└──────────────────────────────────────────────────┘
```

### 6. GAGNÉ'S NINE EVENTS OF INSTRUCTION
```
1. GAIN ATTENTION             → Hook learner interest
   ↓
2. INFORM OBJECTIVES          → State learning goals
   ↓
3. RECALL PRIOR LEARNING      → Activate prerequisites
   ↓
4. PRESENT CONTENT            → Deliver new information
   ↓
5. PROVIDE GUIDANCE           → Examples & demonstrations
   ↓
6. ELICIT PERFORMANCE         → Practice opportunities
   ↓
7. PROVIDE FEEDBACK           → Specific, timely response
   ↓
8. ASSESS PERFORMANCE         → Measure achievement
   ↓
9. ENHANCE RETENTION/TRANSFER → Apply in new contexts
```

### 8. ASSESSMENT TYPES & TIMING
```
FORMATIVE ASSESSMENT              SUMMATIVE ASSESSMENT
(During Learning)                 (After Learning)
─────────────────────────────────────────────────────
• Knowledge checks                • Final projects
• Practice problems               • Portfolios
• Concept maps                    • Comprehensive exams
• Self-assessment                 • Presentations
• Exit tickets                    • Capstone demonstrations
• Peer review                     • Performance tasks

PURPOSE: Monitor & guide          PURPOSE: Evaluate mastery
STAKES: Low, multiple attempts    STAKES: High, few attempts
FEEDBACK: Immediate, detailed     FEEDBACK: Summative, final
USE: Adjust instruction          USE: Credential, grade, certify
```

### 9. COGNITIVE LOAD MANAGEMENT
```
INTRINSIC LOAD          EXTRANEOUS LOAD         GERMANE LOAD
(Topic complexity)      (Poor design)           (Schema building)
───────────────────────────────────────────────────────────────
Cannot be reduced       CAN be reduced          Should be maximized

• Inherent difficulty   • Confusing layout      • Deep processing
• Number of elements    • Irrelevant info       • Integration
• Interactivity        • Split attention       • Automation
                       • Redundancy            • Pattern recognition

STRATEGIES TO MANAGE:
1. Chunk information (5-9 items)
2. Use worked examples
3. Provide scaffolding
4. Build from simple → complex
5. Allow processing time
```

### 10. SCAFFOLDING PROGRESSION
```
HIGH SUPPORT ════════════════════════════════► LOW SUPPORT
SIMPLE TASKS ════════════════════════════════► COMPLEX TASKS

Phase 1: MODELING
  → Instructor demonstrates, learner observes

Phase 2: GUIDED PRACTICE
  → Learner attempts with immediate support

Phase 3: COLLABORATIVE PRACTICE
  → Learners work together with oversight

Phase 4: INDEPENDENT PRACTICE
  → Learner performs autonomously

Phase 5: TRANSFER & APPLICATION
  → Apply to novel contexts, create new solutions
```

## IMPLEMENTATION WORKFLOW

### Step 1: Map Your Knowledge Graph Structure
- Identify all topics and their hierarchical relationships
- Define prerequisite chains
- Establish competency pathways

### Step 2: Define Competencies & Performance Indicators
- Start with graduation competencies (3-6)
- Break down into content competencies (3-6 per area)
- Define performance indicators (5-10 per competency)

### Step 3: Apply Template to Each Topic Node
- Use the 10-component framework
- Complete all checklist items
- Link to related nodes in graph

### Step 4: Validate & Refine
- Ensure backward alignment (assessments → objectives → activities)
- Check cognitive progression across topics
- Verify all prerequisite relationships

### Step 5: Build Learning Paths
- Generate personalized paths based on competency goals
- Support adaptive learning based on mastery evidence
- Enable multiple entry points and progression rates
