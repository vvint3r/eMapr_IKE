### 3. BLOOM'S TAXONOMY (Revised 2001)
```
        ┌─────────────┐
        │   CREATE    │  ← Highest Order (Design, Construct)
        ├─────────────┤
        │  EVALUATE   │  ← (Critique, Judge)
        ├─────────────┤
        │   ANALYZE   │  ← (Compare, Organize)
        ├─────────────┤
        │    APPLY    │  ← (Implement, Execute)
        ├─────────────┤
        │ UNDERSTAND  │  ← (Explain, Summarize)
        ├─────────────┤
        │  REMEMBER   │  ← Lowest Order (Recall, Recognize)
        └─────────────┘
```

### 4. SOLO TAXONOMY (Depth of Understanding)
```
Level 1: PRESTRUCTURAL
  → Lacks understanding, misses the point

Level 2: UNISTRUCTURAL
  → Identifies one relevant aspect

Level 3: MULTISTRUCTURAL
  → Identifies multiple relevant aspects (disconnected)

Level 4: RELATIONAL
  → Integrates parts into coherent whole

Level 5: EXTENDED ABSTRACT
  → Generalizes to new domains, creates new ideas
```

### 6. GAGNÉ'S NINE EVENTS OF INSTRUCTION
```
1. GAIN ATTENTION             → Hook learner interest
   ↓
2. INFORM OBJECTIVES          → State learning goals
   ↓
3. RECALL PRIOR LEARNING      → Activate prerequisites
   ↓
4. PRESENT CONTENT            → Deliver new information
   ↓
5. PROVIDE GUIDANCE           → Examples & demonstrations
   ↓
6. ELICIT PERFORMANCE         → Practice opportunities
   ↓
7. PROVIDE FEEDBACK           → Specific, timely response
   ↓
8. ASSESS PERFORMANCE         → Measure achievement
   ↓
9. ENHANCE RETENTION/TRANSFER → Apply in new contexts
```

### 8. ASSESSMENT TYPES & TIMING
```
FORMATIVE ASSESSMENT              SUMMATIVE ASSESSMENT
(During Learning)                 (After Learning)
─────────────────────────────────────────────────────
• Knowledge checks                • Final projects
• Practice problems               • Portfolios
• Concept maps                    • Comprehensive exams
• Self-assessment                 • Presentations
• Exit tickets                    • Capstone demonstrations
• Peer review                     • Performance tasks

PURPOSE: Monitor & guide          PURPOSE: Evaluate mastery
STAKES: Low, multiple attempts    STAKES: High, few attempts
FEEDBACK: Immediate, detailed     FEEDBACK: Summative, final
USE: Adjust instruction          USE: Credential, grade, certify
```

### 9. COGNITIVE LOAD MANAGEMENT
```
INTRINSIC LOAD          EXTRANEOUS LOAD         GERMANE LOAD
(Topic complexity)      (Poor design)           (Schema building)
───────────────────────────────────────────────────────────────
Cannot be reduced       CAN be reduced          Should be maximized

• Inherent difficulty   • Confusing layout      • Deep processing
• Number of elements    • Irrelevant info       • Integration
• Interactivity        • Split attention       • Automation
                       • Redundancy            • Pattern recognition

STRATEGIES TO MANAGE:
1. Chunk information (5-9 items)
2. Use worked examples
3. Provide scaffolding
4. Build from simple → complex
5. Allow processing time
```

### 10. SCAFFOLDING PROGRESSION
```
HIGH SUPPORT ════════════════════════════════► LOW SUPPORT
SIMPLE TASKS ════════════════════════════════► COMPLEX TASKS

Phase 1: MODELING
  → Instructor demonstrates, learner observes

Phase 2: GUIDED PRACTICE
  → Learner attempts with immediate support

Phase 3: COLLABORATIVE PRACTICE
  → Learners work together with oversight

Phase 4: INDEPENDENT PRACTICE
  → Learner performs autonomously

Phase 5: TRANSFER & APPLICATION
  → Apply to novel contexts, create new solutions
```

# FRAMEWORK 2

---

Learning a profession can be broken into a set of repeatable components that apply whether the field is analytics management, medicine, law, or design. These components span what to learn (knowledge and skills), how to learn (tutorials, projects, feedback), and how to organize the journey (frameworks, roadmaps, tools, and assessment).[1][2]

## Top-Level Pillars

At the highest level, a profession can be learned through seven integrated pillars: competencies, knowledge assets, learning experiences, tools and environment, assessment and feedback, career navigation, and identity and meta-skills.[3][1]

- Competency model: Defines the skills, behaviors, and performance standards for the role.[4][5]
- Knowledge assets: Concepts, definitions, and reference materials that encode the “theory” of the field.[2][6]
- Learning experiences: Tutorials, practice, case work, and projects that build capability.[7][8]
- Tools and environment: Software, platforms, and real or simulated workplaces.[9][3]
- Assessment and feedback: Reviews, exams, portfolios, and coaching.[10][11]
- Career navigation: Roadmaps, milestones, and professional development plans.[12][3]
- Identity and meta-skills: Professionalism, ethics, and self-directed learning habits.[13][14]

## Competencies And Skills

A competency framework describes what “being good” looks like, usually across levels from novice to expert.[5][4]

- Knowledge: Domain facts, concepts, methods, and regulations relevant to the profession.[1]
- Technical (hard) skills: Tools, methodologies, and procedures specific to the role, such as analytics techniques or coding.[3]
- Functional competencies: Skills tied to particular functions like product, finance, or operations.[10]
- Behavioral (soft) skills: Communication, teamwork, leadership, problem-solving, and adaptability.[13][3]
- Meta-competencies: Learning how to learn, systems thinking, and innovation that transfer across roles.[14][4]

## Knowledge: Concepts, Language, Semantics

Every profession has a conceptual “language” that must be learned to interpret and create work in that domain.[2]

- Core concepts and principles: Foundational ideas (e.g., probability, causality, markets, clinical reasoning) that organize the field.[1]
- Definitions, jargon, and keywords: Terms, abbreviations, and phrases enabling precise communication with peers.[2]
- Frameworks and models: Reference structures such as competency models, learning frameworks, and process models that guide thinking.[15][16]
- Standards and ethics: Codes, regulations, and norms that define responsible practice.[12]

## Frameworks, Models, And Systems

Structured frameworks help organize both how to learn and how to practice.[6][7]

- Learning design models: ADDIE (Analysis, Design, Development, Implementation, Evaluation) and similar systems that structure training.[8][7]
- Competency frameworks: Hierarchical models describing core, functional, and leadership competencies across levels.[4][15]
- Process frameworks: Stepwise methods for doing the work (e.g., problem definition → analysis → recommendation → implementation).[17]
- Blueprint templates: Standardized outlines covering goals, audience, content, activities, and assessments.[6]

## Learning Experiences: Tutorials, Practice, Cases

Practical learning experiences convert abstract knowledge into usable skill.[2]

- Tutorials and step-by-step guides: Sequenced instructions that demonstrate specific tasks or tools.[17]
- Learning activities and drills: Short, focused exercises for individual skills (e.g., modeling, presentations, negotiations).[2]
- Case studies: Narrative problem situations that mirror real practice and require judgment.[2]
- Projects and simulations: End-to-end work products or simulated environments that force integration of multiple skills.[7]
- Reflection activities: Journals, debriefs, and after-action reviews that consolidate learning.[7]

## Tools, Software, And Environment

Tools shape how professionals actually work and therefore must be part of any learning plan.[9][3]

- Core software and platforms: Domain-specific tools (e.g., analytics suites, design software, practice management tools).[3]
- Knowledge and collaboration systems: Learning management systems, documentation platforms, and communication tools.[9][3]
- Practice environments: Sandboxes, labs, or test accounts that allow risk-free experimentation.[9]
- Support tools: Note-taking apps, task managers, and automation utilities that enable productivity and organization.[3]

## Roadmaps, Guides, And Progression

Roadmaps coordinate learning across time so effort compounds toward mastery.[1][3]

- Tiered skill levels: Foundational, intermediate, and advanced layers, often represented as pyramids of general-to-specific competencies.[1]
- Professional development plans: Individual plans that map current competence to future goals with specific actions and timelines.[3]
- Milestones and checkpoints: Role titles, certifications, or responsibilities that signal progress (e.g., associate → manager → director).[12]
- Learning sequences: Ordered paths (e.g., fundamentals → tools → projects → specialization) that manage cognitive load.[6][7]

## Assessment, Feedback, And Evaluation

Deliberate assessment closes the loop between effort and improvement.[11][10]

- Diagnostic assessment: Upfront tests or portfolio reviews to identify starting level and gaps.[11]
- Formative assessment: Ongoing quizzes, peer reviews, and coaching during learning activities.[17]
- Summative assessment: Exams, capstone projects, or performance reviews that certify outcomes.[11]
- Behavioral indicators: Observable actions that show competency for each level of skill.[10]
- Feedback mechanisms: Structured critique, performance dashboards, and 1:1 reviews that drive growth.[18]

## Hard Skills Versus Soft Skills

Both types of skills are necessary, but they are learned and assessed differently.[13][3]

- Hard skills: Tool operation, specific methods, data analysis techniques, design patterns, or legal procedures.[3]
- Soft skills: Communication, leadership, conflict resolution, emotional intelligence, and cultural competence.[19][13]
- Context of practice: Soft skills are usually learned through social interaction, mentoring, and real work; hard skills through drills, projects, and structured practice.[19][3]

### Skills Category Overview

| Category             | Examples (generic)                               | Typical Learning Methods                                |
|----------------------|--------------------------------------------------|---------------------------------------------------------|
| Technical/Functional | Tools, methods, domain procedures                | Tutorials, labs, guided projects, certifications[3] |
| Cognitive            | Critical thinking, analysis, decision-making     | Case studies, problem sets, scenarios[3][2]    |
| Interpersonal        | Communication, teamwork, negotiation             | Role-plays, group work, coaching[13][19]          |
| Intrapersonal        | Self-management, resilience, time management     | Reflection, habit systems, mentoring[3][14]     |
| Leadership           | Vision, delegation, stakeholder management       | Stretch roles, feedback, leadership programs[13][3] |

## Supporting Elements: Community, Identity, And Meta-Learning

A profession is also a community and identity, not just a set of tasks.[12][13]

- Professional identity: Seeing oneself as part of a discipline with shared values and standards.[12]
- Communities of practice: Peer groups, professional associations, and online communities that share tacit knowledge.[19][9]
- Mentorship and apprenticeship: Relationships with more experienced practitioners that accelerate learning.[18]
- Active learning strategies: Techniques such as deliberate practice, spaced repetition, and self-testing that make learning more efficient.[14]

## Putting It Together As A Hierarchy

A generic hierarchical structure for learning any profession can be organized as:

1. Vision and outcomes  
   - Professional role definition, career goals, and ethical standards.[12]

2. Competency framework  
   - Knowledge areas, technical skills, behavioral skills, and meta-competencies.[15][4]

3. Roadmap and stages  
   - Tiered levels, learning sequences, milestones, and development plans.[1][3]

4. Curriculum and content

A hierarchical learning roadmap starts from outcomes and roles at the top, then breaks down into competencies, skills, and finally concrete learning activities and resources at the bottom.[1][2]

Below is a reusable 6-level structure you can apply to any profession.

***

## 1. Define Destination and Scope

Begin by clarifying what “success” in the profession looks like and which roles the roadmap covers.[2][3]

- Specify target roles and levels (e.g., Junior → Mid → Senior → Manager).[4][2]
- Write outcome statements per level (what someone in that role can reliably do and deliver).[5][2]

***

## 2. Build a Competency and Proficiency Matrix

Next, describe the profession as a set of core competencies, each with levels of mastery.[6][2]

- Identify 6–10 core competencies (e.g., technical, domain, communication, leadership, ethics).[7][2]
- For each competency, define 3–6 proficiency levels (Beginner → Developing → Competent → Proficient → Advanced → Expert) with observable behaviors.[8][6]

**Example structure (conceptual):**

| Dimension       | Beginner                                  | Competent                                  | Expert                                      |
|----------------|--------------------------------------------|--------------------------------------------|---------------------------------------------|
| Core Technical | Follows recipes with guidance [2]     | Independently solves common problems [2] | Designs novel solutions and standards [6] |
| Communication  | Conveys findings when prompted [1]    | Adapts message to audience [1]        | Shapes decisions and strategy via narrative [4] |

***

## 3. Map Skills and Knowledge to Each Level

Translate each competency level into specific skills, knowledge, and outputs.[9][1]

- For each competency × level, list:  
  - Knowledge blocks (concepts, frameworks, domain facts).[10][9]
  - Skills/tasks (what they must be able to do on the job).[1][2]
- Distinguish foundational vs advanced vs specialized skills to preserve hierarchy.[3][1]

***

## 4. Sequence Learning Phases and Milestones

Group skills into time-bound phases so progression is clear and paced.[11][12]

- Define phases such as Foundation → Development → Mastery, each with duration and focus.[13][11]
- Attach milestones per phase (e.g., “Complete project X”, “Run Y independently”, “Mentor a junior on Z”).[14][15]

**Example phased view:**

| Phase       | Focus                                  | Typical Outputs                          |
|------------|-----------------------------------------|------------------------------------------|
| Foundation | Core concepts, basic tools [11]     | Guided exercises, small tasks [11]   |
| Development| Complex tasks, autonomy [14]        | End-to-end projects [14]            |
| Mastery    | Strategy, leadership, innovation [1]| Mentoring, system design [1]        |

***

## 5. Attach Learning Activities, Resources, and Assessments

At the bottom of the hierarchy, connect concrete learning experiences and checks to each milestone.[16][5]

- Choose learning methods per skill (tutorials, case studies, projects, mentoring, courses).[16][14]
- Define assessment types: quizzes for knowledge, performance tasks and portfolios for skills, 360 or manager feedback for behaviors.[17][2]

***

## 6. Personalize, Review, and Evolve the Roadmap

Finally, treat the roadmap as living: adapt it to individuals and update it as the profession changes.[9][10]

- Use self- and manager assessments to locate a person in the matrix and generate a personalized path.[11][1]
- Revisit competencies and phases periodically to reflect new tools, methods, and emerging skills in the profession.[18][10]
