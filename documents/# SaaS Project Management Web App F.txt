## 12. Integration Guide for CSV Files

### How These CSVs Work Together:

1. **Master Project Tracker** → Links to all other files via project_id
2. **OKR Tracker** → References project_id to show which OKRs align with which projects
3. **Sprint Tracker** → Tracks velocity for capacity planning
4. **User Stories** → References sprint_id and feeds into sprint planning
5. **Risk Register** → Monitors project risks that could affect OKRs
6. **Growth Metrics** → Measures KRs from OKR tracker
7. **Team Capacity** → Ensures realistic sprint planning
8. **Feature Priority** → Feeds into user story creation and sprint planning
9. **Budget Tracker** → Ensures projects stay within financial constraints
10. **Stakeholder Log** → Tracks communication effectiveness
11. **Quality Metrics** → Ensures technical KRs are met

### Weekly Workflow Using These CSVs:

#### Monday - Planning Review
1. Update `team_capacity.csv` for the week
2. Review `risk_register.csv` for new/changed risks
3. Check `budget_tracker.csv` burn rate

#### Tuesday - Sprint Check-in
1. Update `sprint_tracker.csv` with progress
2. Update `user_stories.csv` status
3. Log blockers in risk register if needed

#### Wednesday - Metrics Review
1. Update `growth_metrics.csv` with latest data
2. Update `quality_metrics.csv` from monitoring tools
3. Calculate OKR progress in `okr_tracker.csv`

#### Thursday - Stakeholder Prep
1. Prepare updates from all trackers
2. Update `stakeholder_log.csv` with communications
3. Update `master_project_tracker.csv` health status

#### Friday - Retrospective & Planning
1. Complete sprint data if ending
2. Update `feature_priority.csv` based on feedback
3. Plan next week's capacity

### Automation Tips:

1. **Use Google Sheets or Excel** with these features:
   - VLOOKUP to connect data between sheets
   - Conditional formatting for status colors
   - Data validation for consistent entries
   - Pivot tables for reporting

2. **Create a Dashboard** that pulls from these CSVs:
   - Overall project health (from master tracker)
   - OKR progress bars
   - Sprint velocity trend
   - Budget burn-down chart
   - Risk heat map

3. **Set up Alerts** for:
   - Budget overruns
   - High-risk scores
   - Missed OKR targets
   - Low quality metrics

### Sample Python Script for CSV Analysis:

```python
import pandas as pd
import datetime

# Load all CSV files
master = pd.read_csv('master_project_tracker.csv')
okrs = pd.read_csv('okr_tracker.csv')
sprints = pd.read_csv('sprint_tracker.csv')
stories = pd.read_csv('user_stories.csv')

# Calculate overall project health
def calculate_health_score():
    # Check OKR progress
    okr_avg = okrs['progress_%'].mean()
    
    # Check sprint velocity
    completed_sprints = sprints[sprints['sprint_status'] == 'Completed']
    velocity_trend = completed_sprints['velocity'].iloc[-3:].mean()
    
    # Check story completion rate
    completion_rate = (stories[stories['status'] == 'Completed'].shape[0] / 
                      stories[stories['status'] != 'Not Started'].shape[0] * 100)
    
    health_score = (okr_avg * 0.4 + completion_rate * 0.4 + 
                   (velocity_trend / sprints['story_points_committed'].mean() * 100) * 0.2)
    
    return health_score

# Generate weekly report
def generate_weekly_report():
    report = {
        'Week': datetime.date.today().isocalendar()[1],
        'Overall Health': calculate_health_score(),
        'Active Projects': master[master['status'] == 'In Progress'].shape[0],
        'OKR Progress': okrs['progress_%'].mean(),
        'Sprint Velocity': sprints[sprints['sprint_status'] == 'In Progress']['velocity'].iloc[0],
        'Open Risks': risks[risks['status'] == 'Active'].shape[0],
        'Budget Utilization': (budget['spent_to_date'].sum() / budget['allocated_budget'].sum() * 100)
    }
    return report
```
