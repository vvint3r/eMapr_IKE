# Year 1: Infrastructure, Trust, and Descriptive Analytics

## Phase 1: The Audit, Architecture, and Hygiene (Months 1–3)
**Theme:** *Garbage In, Garbage Out mitigation.*
**Key Stakeholders:** Engineering, Legal/Privacy, Web Dev.

*   **1.1 The Great Tracking Audit (Client-Side)**
    *   **1.1.1 Tag Management System (TMS) Governance**
        *   **Container Structure:** Audit of Google Tag Manager (GTM) / Tealium.
            *   *Cleanup:* Removal of "Ghost Tags" (tags from fired agencies or deprecated tools).
            *   *Trigger Logic:* moving from URL-based triggers (unreliable) to `customEvent` triggers pushed by the data layer.
        *   **The Data Layer Object (DLO):** Defining a standardized JSON schema.
            *   *E-commerce Standard:* `transaction_id`, `value`, `tax`, `shipping`, `currency`, `items[]`.
            *   *User Scope:* `user_id` (hashed), `customer_lifetime_value` (bucketed), `login_status`.
            *   *Page Scope:* `page_type` (PLP vs PDP), `author`, `publish_date`.
    *   **1.1.2 Cookie Consent & Legal Compliance (The Risk Shield)**
        *   **CMP Integration:** Ensure OneTrust/Cookiebot actually blocks scripts before consent (checking `GCS` parameter in GA4).
        *   **PII Redaction:** Implementation of regex filters to scrub email addresses/ZIP codes accidentally passing through URL parameters (a common GDPR violation).
    *   **1.1.3 Cross-Domain & App Tracking**
        *   **Measurement Protocol:** Configuring tracking across `marketing-site.com` $\rightarrow$ `app.product.com`.
        *   **Unification:** Linking Firebase/Mobile SDK data streams with Web streams for a single user view.

*   **1.2 Modern Data Stack (MDS) Implementation**
    *   **1.2.1 Extraction & Loading (ELT)**
        *   **Pipeline Selection:** Fivetran or Airbyte deployment.
        *   **Connector Configuration:**
            *   *Ad Platforms:* FB Ads, Google Ads, LinkedIn, TikTok (pulling spend, impressions, clicks).
            *   *Transactional:* Stripe, Shopify, or Salesforce data extraction.
            *   *Email/SMS:* Klaviyo/Hubspot (pulling campaign send/open data).
    *   **1.2.2 Data Warehousing (The Central Brain)**
        *   **Schema Architecture:**
            *   *Bronze Layer:* Raw JSON ingestion (immutable).
            *   *Silver Layer:* Cleaned, de-duped, normalized tables (handling currency conversion here).
            *   *Gold Layer:* Business-ready tables (e.g., `fact_daily_spend`, `dim_customer`).

*   **1.3 Taxonomy & Standardization**
    *   **1.3.1 The UTM Consensus**
        *   Enforcing snake_case for all parameters (no `Email` vs `email`).
        *   Standardizing `utm_source` vs. `utm_medium` (e.g., ensuring 'facebook' is the source and 'cpc' is the medium, not vice versa).
    *   **1.3.2 Campaign Naming Convention**
        *   Syntax: `Region_Channel_Objective_Date_CreativeID`
        *   *Why:* Allows for regex parsing in the database to auto-categorize spend without manual lookup tables.

## Phase 2: Business Logic, Reporting & Financial Alignment (Months 4–6)
**Theme:** *One Version of the Truth.*
**Key Stakeholders:** CFO, CMO, Sales Leaders.

*   **2.1 Financial Reconciliation (The "Finance Bridge")**
    *   **2.1.1 Definition of "Revenue"**
        *   **Metric Mapping:** Mapping `GA4 Conversion Value` vs. `ERP Recognized Revenue`.
        *   **Variance Analysis:** Calculating the acceptable drift (usually <5%).
        *   **Return Logic:** Handling refunds/chargebacks in analytics (does a return in Month 2 negates credit in Month 1?).
    *   **2.1.2 Cost Calculation**
        *   **The "All-in" Loading:** creating logic to distribute Agency Fees, Tool Costs, and Creative Production costs across paid channels to calculate *True* ROAS, not just Media ROAS.

*   **2.2 Visualization & Dashboard Engineering**
    *   **2.2.1 Dashboard Philosophy (Information Design)**
        *   **Cognitive Load Management:** Removing "chart junk" (3D pie charts, excessive gridlines).
        *   **Preattentive Attributes:** Using color *only* to denote "Good/Bad" performance or specific categorical distinctions, not for decoration.
    *   **2.2.2 The Three-Tiered Dashboard Suite**
        *   **Tier 1: The Board Deck (Strategic):** Blended CAC, LTV:CAC Ratio, YoY Revenue Growth. (Refreshed Monthly).
        *   **Tier 2: The Marketing Pulse (Operational):** Channel-specific breakdown, Weekly Pacing vs. Budget, Lead Velocity. (Refreshed Daily).
        *   **Tier 3: The Exploratory Workbench (Tactical):** High-cardinality tables for drill-downs (Creative ID performance, Keyword performance).

*   **2.3 Baseline Statistical Profiling**
    *   **2.3.1 Control Charting**
        *   Establishing "Normal" variance for traffic and conversion rates using Rolling Means and Standard Deviations ($\pm 2\sigma$).
        *   *Action:* Create automated alerts (Slack/Email) only when metrics breach the $\pm 2\sigma$ threshold (preventing "noise" panic).

## Phase 3: Resilience, Retention & Advanced Segmentation (Months 7–9)
**Theme:** *Signal loss prevention and Customer Depth.*
**Key Stakeholders:** CRM Team, Developers.

*   **3.1 Signal Resilience (Fighting ITP & Cookie Loss)**
    *   **3.1.1 Server-Side Tracking (CAPI)**
        *   **Implementation:** Setup of Meta Conversions API and Google Enhanced Conversions.
        *   **De-duplication Keys:** Ensuring the `event_id` matches on both browser and server events to prevent double counting.
    *   **3.1.2 First-Party Data Enrichment**
        *   **Lead Scoring Models:** Implementing explicit scoring (Job Title = VP +10pts) and implicit scoring (Viewed Pricing Page +5pts).
        *   **Zero-Party Data capture:** Implementing post-purchase surveys ("How did you hear about us?") to validate attribution.

*   **3.2 Advanced Segmentation (Clustering)**
    *   **3.2.1 RFM Analysis (Recency, Frequency, Monetary)**
        *   **Mathematical Model:** Scoring every customer 1-5 on each axis.
        *   **Output:** Segments like "Champions" (5-5-5), "Hibernating" (1-x-5), "Can't Lose Them" (3-5-5).
    *   **3.2.2 K-Means Clustering (Unsupervised Learning)**
        *   Using Python (Scikit-Learn) to find natural groupings in customer behavior beyond spend (e.g., "Window Shoppers," "Discount Hunters," "Power Users").
        *   *Visual:* Elbow Method to determine the optimal number of clusters ($k$).

*   **3.3 Retention Mechanics**
    *   **3.3.1 Cohort Analysis (Triangle Charts)**
        *   **Layering:** Analyze retention not just by "Month of Joining" but by "Acquisition Channel."
        *   *Hypothesis:* "Do customers acquired via TikTok churn faster than those via SEO?"
    *   **3.3.2 Churn Definitions**
        *   **Logo Churn:** $\%$ of customers lost.
        *   **Net Revenue Churn:** Revenue lost - Expansion revenue (Does expansion offset the losses?).

## Phase 4: Testing & Optimization Strategy (Months 10–12)
**Theme:** *Moving from Observation to Action.*
**Key Stakeholders:** Growth Team, Product Design.

*   **4.1 Experimentation Framework (CRO)**
    *   **4.1.1 Statistics of A/B Testing**
        *   **Power Analysis:** Determining sample size before the test starts to avoid "Peeking" bias.
        *   **Significance Thresholds:** Deciding on $\alpha = 0.05$ (95% confidence) vs $\alpha = 0.10$ (90% confidence - for faster, riskier moves).
        *   **SRM Checks:** Sample Ratio Mismatch. If the traffic split isn't exactly 50/50, there is a technical bug.
    *   **4.1.2 Test Hierarchy**
        *   *Macro Tests:* Value Proposition, Pricing tiers (High Impact, High Risk).
        *   *Micro Tests:* Button color, CTA text (Low Impact, High confidence).

*   **4.2 Efficiency & Pacing**
    *   **4.2.1 Budget Pacing Algorithms**
        *   Developing a "Burn Rate" visualizer.
        *   *Logic:* `(Budget - Spend_to_Date) / Days_Remaining` vs `Current_Daily_Run_Rate`.
    *   **4.2.2 Marginal ROAS Analysis**
        *   Differentiating between *Average* ROAS (Historic) and *Marginal* ROAS (The return on the next dollar spent). identifying the "Point of Diminishing Returns."

# Year 2: Predictive Analytics, Causality, and AI

## Phase 5: Causal Inference & Marketing Mix Modeling (Months 13–16)
**Theme:** *Answering "What actually drove the sale?"*
**Key Stakeholders:** Data Science, C-Suite.

*   **5.1 Marketing Mix Modeling (MMM)**
    *   *Concept:* Econometric regression to quantify the impact of offline + online channels + external factors (seasonality, inflation) without needing cookies.
    *   **5.1.1 Variable Selection (Feature Engineering)**
        *   *Media:* Impressions, Spend, GRPs.
        *   *Context:* Holidays, Weather, Competitor Price Index.
        *   *Organic:* Branded Search Volume (proxy for mental availability).
    *   **5.1.2 Mathematical Transformations**
        *   **Adstock (Lag Effect):** Using Geometric or Weibull distributions to model how long an ad "lingers" in a user's mind (e.g., TV has high adstock, Display has low).
        *   **Saturation (Hill Function):** Modeling the "S-Curve" where increased spend yields zero incremental revenue.
    *   **5.1.3 Tools & Execution**
        *   Implementation of open-source libraries: **Facebook Robyn** (R) or **Google Meridian/LightweightMMM** (Python/Jax).

*   **5.2 Multi-Touch Attribution (MTA) (The Tactical View)**
    *   **5.2.1 Markov Chain Modeling**
        *   *Method:* Analyzing the probability of conversion changing based on the removal of a node (channel) from the path.
        *   *Removal Effect:* Calculating exactly how many conversions are lost if "Facebook Awareness" is removed from the mix.
    *   **5.2.2 Shapley Value (Game Theory)**
        *   Assigning credit based on a player's (channel's) average marginal contribution across all possible coalitions (paths).

*   **5.3 Triangulation**
    *   Creating the "Truth Plot": comparing MMM results vs. MTA results vs. Platform Reported results to create a weighted index for budget allocation.

## Phase 6: Experimentation Gold Standard (Months 17–19)
**Theme:** *Proving Incrementality.*

*   **6.1 Geo-Lift Experiments (The Calibrator)**
    *   **6.1.1 Matched Market Design**
        *   Identifying specific geographic regions (e.g., DMA: Phoenix) that statistically correlate highly with a control region (e.g., DMA: Denver).
    *   **6.1.2 Time-Based Regression**
        *   Using **CausalImpact** (Bayesian structural time-series) to predict what "would have happened" in the test region without ads, comparing it to actuals.
    *   **6.1.3 Holdout Groups**
        *   "Ghost Ads" / "Dark Tests": Executing exclusion audiences on Facebook to measure the baseline organic conversion rate of the target demographic.

## Phase 7: Predictive Modeling (Months 20–22)
**Theme:** *Future-proofing revenue.*

*   **7.1 Predictive LTV (pLTV)**
    *   **7.1.1 Buy 'Til You Die (BTYD) Models**
        *   **Pareto/NBD:** Modeling the friction between "Purchase Rate" and "Dropout Rate."
        *   **Gamma-Gamma Model:** Estimating the average monetary value of future transactions.
    *   **7.1.2 Activation**
        *   Creating custom audiences in Ad Platforms defined not by *past* spend, but by *predicted future* spend.

*   **7.2 Propensity Scoring (Classification)**
    *   **7.2.1 Propensity to Buy**
        *   Logistic Regression/Random Forest models scoring non-purchasers on likelihood to convert.
        *   *Tactic:* Suppression. Don't spend ad money on users with >90% propensity (they will buy anyway) or <10% (they won't buy). Focus spend on the "Persuadables" (40-60%).
    *   **7.2.2 Propensity to Churn**
        *   Identifying "at-risk" behaviors (e.g., multiple exports of data, decreased login time).

## Phase 8: Democratization, AI & Ops at Scale (Months 22–24)
**Theme:** *removing the Analyst from the loop.*

*   **8.1 Analytics Engineering as Code**
    *   **8.1.1 dbt (Data Build Tool) Mastery**
        *   Moving all SQL logic into version-controlled (Git) repos.
        *   **Documentation:** Auto-generating dependency graphs (DAGs) so any analyst knows where data comes from.
    *   **8.1.2 Data Quality Automation**
        *   Implementing "Great Expectations" or dbt tests.
        *   *Example:* If `daily_revenue` drops by >20% or if `null` values in `campaign_id` exceed 5%, the pipeline breaks and alerts Engineering before the dashboard updates.

*   **8.2 GenAI for Analytics (The Interface Shift)**
    *   **8.2.1 Text-to-SQL Integration**
        *   Connecting an LLM (OpenAI/Anthropic) via API to the database schema.
        *   *Goal:* allowing the CEO to ask Slack: "What was the CPA for non-branded search in Q3 vs Q2?" and getting an automated, accurate answer.
    *   **8.2.2 Qualitative at Scale**
        *   Using NLP (Natural Language Processing) to analyze 10,000 customer support tickets to find correlation between "specific complaints" and "churn."

# Supporting Pillar: The Organization & Ethics

### Team Structure Evolution
*   **Year 1 Team:** Manager + 1 Analytics Engineer (Pipeline focused) + 1 Generalist Analyst (Dashboard focused).
*   **Year 2 Team:** Add Data Scientist (MMM/Predictive focused) + Analytics Translator (Bridge between stakeholders and quants).

### Ethical Framework
*   **Algorithmic Bias Audits:** Periodically checking if "High Value" models are systematically excluding protected classes (race, gender, zip code proxies).
*   **Privacy Stewardship:** Maintaining a deletion architecture—if a user requests GDPR deletion, does it propagate from the CRM $\rightarrow$ Warehouse $\rightarrow$ Backups?
